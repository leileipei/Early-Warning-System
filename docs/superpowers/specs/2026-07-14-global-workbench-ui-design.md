# SQL 预警系统全站工作台 UI 优化设计

## 目标与范围

将现有后台统一为“清晰分区工作台”风格，提升规则、配置和日志等高频管理页面的扫描效率与移动端可读性。本轮仅调整模板结构和 CSS 视觉表现，不改变路由、字段、表单提交地址、JavaScript 业务逻辑、数据模型或接口。

涉及页面：仪表盘、规则列表及编辑页、规则历史、配置页及其两个编辑页、日志页和登录页。公共基础为 `base.html` 与 `styles.css`。

## 视觉系统

- 页面使用中性浅灰背景与白色内容表面，主操作使用蓝色；成功、警告和失败状态分别使用绿、橙、红，并保持足够对比度。
- 页面最大宽度扩展至适合后台表格的桌面阅读范围；各页采用“上下文说明、标题及主要操作、业务内容”的一致层级。
- 面板仅用于指标、表单、列表和确有边界的工作区域，避免把整页分割成嵌套卡片。
- 统一边框、圆角、阴影克制的表面层级、聚焦态、提示消息、空状态、按钮与表单控件。
- 排版使用固定字号层级和正常字距，不使用随视口变化的字体缩放。

## 公共框架与响应式行为

- 顶部导航保留“仪表盘、规则、配置、日志、退出”以及全部既有链接；通过请求路径标识当前导航项，桌面端紧凑横向展示。
- 主标题区域保留现有正文和操作按钮，但统一间距、按钮优先级和小屏换行规则。
- 桌面端双栏内容和双列字段保持网格布局；小于当前移动断点时收为单列。
- 表格继续放在可横向滚动容器中，确保列不被压缩到不可读；操作组允许换行，但按钮不改变数据或提交行为。
- 登录页保留独立、简洁的认证表单，增强输入焦点、错误状态和品牌层级。

## 页面应用

- 仪表盘：保留现有指标与两栏记录，仅强化指标数字、辅助说明、分区标题和空状态。
- 规则与日志：统一列表标题、筛选区、表头、行间距和操作区；状态信息以可扫描的视觉层级呈现，保留所有既有操作文字与确认提示。
- 配置：SQL Server 与 SMTP 区域保持既有表单和列表顺序，增强区块标题、测试/编辑/删除操作的层次和移动端排版。
- 规则编辑与配置编辑：保留字段、校验和 SQL 检测/预览行为；将字段组、SQL 工作区、反馈区和提交按钮做成更清晰的连续工作流。
- 规则历史：沿用表格数据，改善版本详情、元数据和空状态的阅读层级。

## 可访问性与兼容性

- 保留现有语义化表单、标签、状态区域和 CSRF 字段。
- 所有键盘可聚焦控件保留清晰焦点轮廓；提示和状态色不作为唯一信息载体。
- 不引入新的前端依赖、远程字体、图标库或图片资源。

## 验收与验证

- 所有既有模板渲染测试、路由测试和 SQL 检测/预览交互继续通过。
- 主要桌面页面在常见宽度下无溢出、按钮重叠或标题遮挡；窄屏下导航、标题操作、表单网格和表格容器保持可用。
- 规则保存、导入导出、测试连接、SMTP 测试、日志筛选、执行、归档和登录等既有提交地址与方法不变。
- 使用完整 Python 测试套件与 Ruff 检查；通过浏览器检查仪表盘、规则、配置、日志、规则编辑和登录页的桌面与移动布局。
